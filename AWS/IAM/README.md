# AWS IAM লার্নিং গাইড

## IAM কি?
**IAM** = Identity and Access Management (পরিচয় এবং অ্যাক্সেস ব্যবস্থাপনা)

AWS এ কে কি কাজ করতে পারবে তা নির্ধারণ করে IAM। এটি AWS এর নিরাপত্তার সবচেয়ে গুরুত্বপূর্ণ অংশ।

## IAM এর মূল ধারণা

### 1. **User (ব্যবহারকারী)**
একজন মানুষ বা অ্যাপ্লিকেশন যারা AWS ব্যবহার করে।

### 2. **Role (ভূমিকা)**
অনুমতির সমূহ যা একজন ব্যবহারকারী বা সেবা ব্যবহার করতে পারে।

### 3. **Policy (নীতি)**
কি কি কাজ করা যাবে তার নিয়ম। JSON ফর্ম্যাটে লেখা।

### 4. **Permission (অনুমতি)**
নির্দিষ্ট কাজ করার অনুমতি।

---

## নীতির মূল অংশ

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["service:action"],
      "Resource": "*"
    }
  ]
}
```

| অংশ | মানে |
|-----|------|
| Version | নীতির সংস্করণ |
| Statement | অনুমতি/নিষেধের তালিকা |
| Effect | Allow (অনুমতি) বা Deny (নিষেধ) |
| Action | কি কি কাজ করা যাবে |
| Resource | কোন কোন জিনিসে কাজ করা যাবে |

---

## সাধারণ উদাহরণ

### সম্পূর্ণ অ্যাক্সেস
```json
{
  "Effect": "Allow",
  "Action": "service:*",
  "Resource": "*"
}
```
সব কাজ, সব জায়গায়।

### শুধু দেখার অনুমতি
```json
{
  "Effect": "Allow",
  "Action": "service:Describe*",
  "Resource": "*"
}
```
শুধু তথ্য দেখা যায়।

### সম্পূর্ণ নিষেধ
```json
{
  "Effect": "Deny",
  "Action": "service:*",
  "Resource": "*"
}
```
কিছুই করা যায় না।

## IAM এর সুবিধা

✅ নিরাপত্তা - শুধু প্রয়োজনীয় অনুমতি দেওয়া যায়
✅ নিয়ন্ত্রণ - সবাইকে আলাদা অনুমতি দেওয়া যায়
✅ অডিট - কে কি করেছে তা দেখা যায়
✅ বিনামূল্যে - কোন অতিরিক্ত খরচ নেই

---

## শেখার ধাপ

1. **IAM ব্যবহারকারী তৈরি করুন** - AWS কনসোলে গিয়ে নতুন ব্যবহারকারী তৈরি করুন
2. **নীতি বুঝুন** - পলিসি ডকুমেন্ট পড়ুন এবং বুঝুন
3. **অনুমতি দিন** - ব্যবহারকারীকে অনুমতি এটাচ করুন
4. **পরীক্ষা করুন** - ব্যবহারকারী হিসেবে লগইন করে পরীক্ষা করুন

---

## গুরুত্বপূর্ণ টিপস

⚠️ **Root ব্যবহার করবেন না** - সবসময় IAM ব্যবহারকারী ব্যবহার করুন
⚠️ **সর্বনিম্ন অনুমতি নীতি** - শুধু প্রয়োজনীয় অনুমতি দিন
⚠️ **নিয়মিত অডিট** - কাদের কি অনুমতি আছে তা চেক করুন
